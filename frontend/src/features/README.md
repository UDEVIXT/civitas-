## /features
Aquí reside la lógica de negocio dividida por funcionalidades o módulos. Cada subcarpeta representa una característica específica de la app.

### Contenido
Dentro de cada "feature" (ej: auth, products, cart) se encuentran sus propios componentes, hooks y servicios.

Esto permite que la aplicación sea escalable y fácil de navegar al agrupar todo lo relacionado a un solo concepto en un solo lugar.
